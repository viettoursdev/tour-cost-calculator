# PR-3.5 NCC Tab — Design

_Date: 2026-06-04_
_Phase: 3 (tab-by-tab port)_
_Builds on: PR-3.4 Customer (merged at `a0aca65`)_
_Status: Approved design, ready for implementation plan_

---

## 1. Goal

Port the **NCC (Nhà Cung Cấp / Suppliers) tab** from the legacy monolith into the Vite app. Pattern is identical to PR-3.4 Customer with three structural differences: no type toggle, multi-select sectors (13 options with colors), and a location field.

**Non-goals:** Rate-card linkage (legacy NCCView had a "📋 Rate card" button for certain sectors — deferred). Tests.

---

## 2. Differences from PR-3.4 Customer

| Aspect | Customer | NCC |
|---|---|---|
| Extra fields | `type` (company/individual) | `sectors: string[]` (multi-select) + `location: string` |
| Firestore path | `viettours/customer_list.customers[]` | `viettours/ncc_master.suppliers[]` |
| List filter | type | sector |
| Validation | name required | name **AND** at least 1 sector required |
| Permission | `manageCustomers` | `manageNCC` |
| Array key in Firestore doc | `customers` | `suppliers` |
| Color scheme | blue (`#2980b9`) | teal (`#0d7a6a` — primary) |

## 5. Firestore API additions (`src/lib/firebase.ts`)

```ts
const NCC_DOC = doc(db, 'viettours', 'ncc_master');

/**
 * Subscribe to the NCC (suppliers) list. Callback receives the array.
 * Source: legacy window.fbOnNCC reads data.suppliers.
 */
export function fbSubscribeNcc(cb: (list: Ncc[]) => void): Unsubscribe {
  return onSnapshot(NCC_DOC, (snap) => {
    cb(snap.exists() ? ((snap.data().suppliers as Ncc[]) ?? []) : []);
  });
}

/**
 * Full-overwrite push of the NCC list.
 * Source: legacy window.fbPushNCC writes { suppliers, updatedAt, updatedBy }.
 */
export async function fbPushNcc(
  list: Ncc[],
  pushedBy: { name: string; role: string },
): Promise<void> {
  await setDoc(NCC_DOC, {
    suppliers: list,
    updatedAt: new Date().toISOString(),
    updatedBy: `${pushedBy.name} (${pushedBy.role})`,
  });
}
```

## 6. `nccStore` (`src/stores/nccStore.ts`)

Identical shape to `customerStore`. Optimistic update pattern.

```ts
type NccState = {
  suppliers: Ncc[];
  loading: boolean;
  syncing: boolean;
  init: () => Unsubscribe;
  save: (form: Ncc) => Promise<void>;
  delete: (id: string) => Promise<void>;
};
```

`save` distinguishes create vs update by `suppliers.find(s => s.id === form.id)`. Same `useAuthStore.getState().currentUser` cross-store read.

## 7. Components

```
src/components/ncc/
├── constants.ts     # NCC_SECTORS, SECTOR_COLOR
├── NCCView.tsx      # Tab root: header, search, sector filter, card grid, confirm-delete dialog
└── NCCModal.tsx     # Create/edit/view Dialog
```

### `NCCModal`

Fields in order (matching `legacy.html:10043-10104`):
1. **Tên NCC** — `<TextField required>`. Validation: `name.trim() !== ''`.
2. **Lĩnh vực dịch vụ** — row of colored chip-toggle buttons (one per sector from `NCC_SECTORS`). Active sector = filled bg using `SECTOR_COLOR[s]`. Tap toggles in/out of `form.sectors`. Validation: `sectors.length >= 1`.
3. **Địa điểm** — `<TextField>` (optional).
4. **Người liên hệ** — identical to Customer's contact section (same 2×2 grid per contact, add/remove).
5. **Ghi chú** — `<TextField multiline rows={3}>`.

`handleSave` validation:
```ts
if (!form.name.trim()) { window.alert('Vui lòng nhập tên NCC'); return; }
if (form.sectors.length === 0) { window.alert('Vui lòng chọn ít nhất 1 lĩnh vực'); return; }
```

### `NCCView`

Same structure as `CustomerView` with:
- Filter: `<Select>` over NCC_SECTORS + "Tất cả" (instead of type filter).
- Search: `name`, `location`, `note`, all `contacts[*].{name,phone,email,position}`.
- Card content: name + up to 3 sector chips + location (if set) + first 2 contacts + note.
- Permission: `hasPerm(currentUser, 'manageNCC')` for add/edit/delete.
- Confirm delete: inline MUI `<Dialog>` (same pattern as CustomerView).

## 10. Risks

| Risk | Mitigation |
|---|---|
| Sector chips overflow on narrow screens | `flexWrap: 'wrap'` on the chip row (matches legacy behavior). |
| Rate-card link button missing vs legacy | Explicitly deferred to PR-4.x. No user-facing regression yet; the modal will show sectors and link to rate-card can be added later. |
| Optimistic delete of wrong supplier | Same risk as Customer; same fix: inline confirm dialog before calling `del()`. |
