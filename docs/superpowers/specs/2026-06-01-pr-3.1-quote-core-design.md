# PR-3.1 Quote Core — Design

_Date: 2026-06-01_
_Phase: 3 (tab-by-tab port)_
_Project: Viettours Tour Cost Calculator_
_Status: Approved design, ready for implementation plan_

---

## 1. Goal

Port the **Quote tab core** from the legacy single-file React monolith (`public/legacy.html`) into the Vite + TypeScript + MUI + Zustand stack. After this PR, the Vite app's "Báo Giá" tab is fully functional for domestic and international tour quotes; users no longer need the legacy fallback for the Quote workflow.

**Drivers:**

- Quote is the highest-traffic tab and the central workflow Sales/Ops use daily.
- The MUI shell + auth + Rate Card store from Phases 0–2 are stable; PR-2.1 validated the porting pattern. Time to apply it to a bigger surface.
- Cuts a meaningful chunk off the legacy fallback's traffic, narrowing the surface for the next ports.

**Non-goals:**

- DMC view, currency-output selector, DMC compare panel (PR-3.9).
- Cloud quote history, collaborators, SaveQuoteModal (PR-3.2).
- Itinerary / Menu / Visa / DocTranslate sub-apps (separate Phase 3 PRs or out-of-scope entirely).
- Excel / PDF / Word / Invoice export (PR-4.1).
- Contract / Payment / Acceptance modals reachable from the Quote toolbar in legacy (PR-3.6 / 3.7 / 3.8).
- Push notifications on quote events (PR-3.10).

## 2. Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Tab scope | **Cost view + Summary view + local HistPanel only.** No DMC, no cloud history, no exports beyond JSON. | Matches the original Phases 0–2 plan's PR-3.1 boundary. Bigger scopes are explicitly other PRs. |
| Table component | **Plain MUI `<Table>` + controlled `<TextField>` cells.** | Matches HotelModal pattern from PR-2.1; users already trained. `@mui/x-data-grid` fights the legacy table's compound cells (price+currency, qtyMode+qty). |
| Template selection UX | **Full-screen modal gate** mirroring legacy. | Sales/Ops are trained on this. Auto-default + toolbar switcher was the alternative; rejected as too disorienting. |
| Draft persistence | **Auto-save to `localStorage[vte_quote_draft_${user.u}]`** via Zustand `persist` middleware. Per-user scoped. | Matches user expectation that quote work survives refresh. Per-user scope avoids cross-user leak on shared devices. |
| Snapshot list shape | **Per-user namespaced key `vte_q`** (existing legacy shape). Keyed by `user.u`, list capped at 50, FIFO eviction. | Bidirectional compatibility with legacy app during the Phase 3 transition: a snapshot saved in Vite must be loadable in legacy. |
| Snapshot persistence | **Plain `localStorage` helpers in `src/lib/storage.ts`**, NOT Zustand `persist`. | `persist` doesn't naturally support multi-user-namespaced keys without ugly wrappers. |
| Inline-edit pattern | **Always-editable `<TextField>` cells.** No click-to-edit. | MUI TextField is compact enough; consistent with HotelModal. Drops the legacy `ET`/`EN` helper components. |
| Item shape | **Unchanged from legacy** (id, name, note, cur, price, times, qtyMode, customQty, unit, enabled, foc). | Bidirectional Firestore + localStorage compatibility with legacy. |
| Calculation formulas | **Lifted verbatim from `legacy.html:1686-1692` (`calcVND`), `legacy.html:8448-8450` (`subtotal`), and `legacy.html:8494-8498` (totals composition).** Verified during writing-plans (see Section 6 for the exact composition). | Avoids silent price drift between the two apps during the transition window. |
| Rounding direction | **`Math.ceil` applied to the per-pax selling price** (NOT to the grand total). Grand total = `roundedPerPax × pax`. Matches `legacy.html:8497-8498`. | Verified against legacy. The per-pax rounding produces meaningfully different grand totals than rounding the grand total directly. |
| `svcBasis` | **User-editable additive constant in VND**, added BEFORE margin in the totals chain. Exposed as a `<TextField>` in the CostTotals sidebar. | Legacy treats `svcBasis` as a real input (`legacy.html:8540` passes `setSvcBasis` to SummaryView). The spec initially planned to defer this as a "legacy compat flag"; verification shows it's part of the active math. |
| Sub-views | **`'cost' | 'summary'` only** in PR-3.1. View state lives in store (not component-local) so snapshot loads preserve user position. | PR-3.2 will add `'history'` without restructuring. |
| Tests | **Out of scope.** | Matches the Phases 0–2 spec convention. Calculator math is pure and trivially testable later. |

```ts
  | 'flight' | 'hotel' | 'transport' | 'meal' | 'sight' | 'meeting'

  | 'visa' | 'dmc' | 'service_fee';

export type Template = 'domestic' | 'intl' | 'dmc';

export type QtyMode = 'per_pax' | 'per_group' | 'custom';

export type Item = {
  id: number;
  name: string;
  note: string;
  cur: string;         // currency code; matches keys of QuoteDraft.rates
  price: number;
  times: number;
  qtyMode: QtyMode;
  customQty: number;   // used only when qtyMode === 'custom'
  unit: string;        // one of UNITS constants
  enabled: boolean;
  foc: boolean;
};

export type QuoteInfo = {
  name: string;
  dest: string;
  days: number;
  nights: number;
  startDate: string | null;  // ISO date or null
};

export type QuoteDraft = {
  template: Template | null;             // null = no draft yet, show selector
  info: QuoteInfo;
  pax: number;
  rates: Record<string, number>;         // currency → VND rate (quote-local, overrides master rate-card)
  margin: number;                        // profit margin %
  vat: number;                           // VAT %
  svcBasis: number;                      // user-editable additive constant in VND (added before margin)
  rounding: number;                      // rounding step in VND (e.g. 100000)
  items: Partial<Record<CategoryId, Item[]>>;
  catEnabled: Record<CategoryId, boolean>;
  currentQuoteId: string | null;         // for cloud history (set by PR-3.2; PR-3.1 just preserves the field)
};

export type Snapshot = {
  id: number;                            // local id (Date.now())
  cloudId: string;                       // reserved for PR-3.2; generated at save time
  name: string;
  date: string;                          // vi-VN locale string at save time
  savedBy: string;                       // user.name at save time
  state: QuoteDraft;                     // deep-cloned
};

export type SavedQuotesByUser = Record<string, Snapshot[]>;  // localStorage shape under key `vte_q`
```

### 4.2 `quoteStore` (`src/stores/quoteStore.ts`)

```ts
type QuoteState = {
  draft: QuoteDraft;
  view: 'cost' | 'summary';
  snapshots: Snapshot[];            // current user's snapshots only

  // lifecycle
  init: (user: User) => void;       // scopes persist key to user, restores draft, loads snapshots
  reset: () => void;                // logout hook

  // template + view
  newDraft: (template: Template) => void;
  abandon: () => void;
  setView: (v: 'cost' | 'summary') => void;

  // top-level
  patchInfo: (patch: Partial<QuoteInfo>) => void;
  setPax: (n: number) => void;
  setRate: (cur: string, rate: number) => void;
  setMargin: (n: number) => void;
  setVat: (n: number) => void;
  setSvcBasis: (n: number) => void;
  setRounding: (n: number) => void;

  // items
  toggleCat: (cid: CategoryId) => void;
  addItem: (cid: CategoryId, override?: Partial<Item>) => void;
  updItem: (cid: CategoryId, item: Item) => void;
  delItem: (cid: CategoryId, id: number) => void;

  // JSON export/import
  exportJSON: () => string;                                          // serialized payload with _meta
  importJSON: (raw: string) => { ok: true } | { ok: false; error: string };

  // snapshots
  saveSnapshot: (name: string) => Snapshot;
  loadSnapshot: (id: number) => void;                                // also setView('cost')
  deleteSnapshot: (id: number) => void;
  renameSnapshot: (id: number, name: string) => void;
};
```

**Persistence layout:**

- **Active draft**: `localStorage[vte_quote_draft_${user.u}]`. Per-user scoped. Auto-saved via Zustand `persist({ name: ..., partialize: s => ({ draft: s.draft }) })`. `view` and `snapshots` are NOT persisted via `persist`; `view` is session-only, `snapshots` use their own storage path.
- **Snapshots**: `localStorage[vte_q]`, shape `SavedQuotesByUser` keyed by `user.u`. Read/written via `readSavedQuotes` / `writeSavedQuotes` helpers in `src/lib/storage.ts`. Each save/delete/rename reads the full map, mutates the current user's slice, writes back.
- Maximum **50 snapshots per user** (FIFO eviction on save). Matches legacy `HistPanel.doSave` at `legacy.html:2679`.

**Cross-store reads:**

- `rates` field lives **inside `QuoteDraft`**, NOT in `rateCardStore`. Legacy treats currency-conversion rates as quote-local: a user can override USD=25400 for a specific quote without changing the master rate card. Matches `legacy.html:8371`.
- `pushedBy` for snapshot.savedBy comes from `useAuthStore.getState().currentUser` at save time. No reactive coupling.

**`init(user)` behavior:**

1. Configure the Zustand `persist` middleware key to `vte_quote_draft_${user.u}` (achieved by recreating the persist instance on first init — see implementation plan for the mechanic).
2. Hydrate `draft` from that key. If missing, `draft.template = null` (triggers selector on render).
3. Read full `SavedQuotesByUser` from `vte_q`; set `snapshots` to the current user's slice (or `[]` if missing).

**`reset()`:** called on logout. Clears in-memory state to defaults; does NOT delete localStorage (other users' data must survive).

## 6. Calculator math

```ts
// src/components/quote/calc.ts (excerpt; see plan for full implementation)

export function calcVND(item: Item, rates: Record<string, number>, pax: number): number {
  if (!item.enabled || item.foc) return 0;
  const r = rates[item.cur] ?? 1;
  const qty = item.qtyMode === 'per_pax' ? pax
            : item.qtyMode === 'per_group' ? 1
            : item.customQty;
  return item.price * r * item.times * qty;
}

export function subtotal(draft: QuoteDraft): number {
  return CATS.reduce((s, cat) => {
    if (!draft.catEnabled[cat.id]) return s;
    const items = draft.items[cat.id] ?? [];
    return s + items.reduce((ss, it) => ss + calcVND(it, draft.rates, draft.pax), 0);
  }, 0);
}

export type Totals = {
  totalCost: number;        // = subtotal(draft); kept under legacy name for clarity
  totalProfit: number;      // Math.round((totalCost + svcBasis) × margin/100)
  totalVAT: number;         // Math.round((totalCost + svcBasis + totalProfit) × vat/100)
  sellingPPax: number;      // (totalCost + svcBasis + totalProfit + totalVAT) / pax, unrounded
  roundedPPax: number;      // Math.ceil(sellingPPax / rounding) × rounding  — the per-pax sticker price
  grandTotal: number;       // = roundedPPax × pax  (derived for display)
};

export function computeTotals(draft: QuoteDraft): Totals {
  const totalCost = subtotal(draft);
  const totalProfit = Math.round((totalCost + draft.svcBasis) * draft.margin / 100);
  const totalVAT = Math.round((totalCost + draft.svcBasis + totalProfit) * draft.vat / 100);
  const sellingPPax = draft.pax > 0
    ? (totalCost + draft.svcBasis + totalProfit + totalVAT) / draft.pax
    : 0;
  const step = draft.rounding || 1;
  const roundedPPax = Math.ceil(sellingPPax / step) * step;
  const grandTotal = roundedPPax * draft.pax;
  return { totalCost, totalProfit, totalVAT, sellingPPax, roundedPPax, grandTotal };
}
```

**Verification (completed during writing-plans, lifted verbatim from `legacy.html:8494-8498`):**

1. **`calcVND`** matches `legacy.html:1686-1692`.
2. **`subtotal`** = legacy's `totalCost`, matches the `useMemo` reduce at `legacy.html:8448-8450`.
3. **`computeTotals`** composition: `svcBasis` added BEFORE margin; margin and VAT both `Math.round`-ed to integer VND; **rounding is applied to per-pax, NOT to grand total**; `grandTotal` is the derived display value `roundedPPax × pax`. This produces meaningfully different numbers than rounding the grand total directly — preserving the per-pax rounding behavior is required for cutover price parity.
4. **`Math.ceil`** is the legacy rounding direction (`legacy.html:8498`), confirmed.

**Display formatting:**

- `fmtVND(n)` = `Math.round(n).toLocaleString('vi-VN') + ' ₫'`. Matches legacy `fmt`.
- `fmtCurrency(n, code)` uses `Intl.NumberFormat` for non-VND. Matches legacy `fmtCurrency` at `legacy.html:3698`.

**CostTotals UI layout** (right rail in `<CostView>`):

## 7. HistPanel (local snapshots)

**Storage shape:** `localStorage[vte_q] = SavedQuotesByUser`. Per-user keyed list of `Snapshot`. Max 50 per user, FIFO eviction.

**`Snapshot.cloudId` field:** generated at save time using legacy's algorithm — `Date.now().toString(36) + Math.random().toString(36).slice(2,5)`. PR-3.1 stores it but does not push to Firestore. PR-3.2 picks it up and writes to `quote_projects/{cloudId}` without needing a schema migration.

**UI:**

- `<Drawer variant="permanent" anchor="right">` on `md+` screens. On `sm`, replaced by an icon-toggle in the toolbar that slides the drawer in.
- Top: `[+ Lưu báo giá hiện tại]` button → opens `<Dialog>` with `<TextField>` for name (default = `draft.info.name`). Submit → `saveSnapshot(name)`.
- List of `<ListItem>` per snapshot: name + savedBy + date + `[📂 Load]` + `[✕ Delete]` actions.
- Click `[📂]` → confirm dialog ("Replace current draft?") → `loadSnapshot(id)` → `setView('cost')`.
- Click `[✕]` → confirm dialog ("Delete this snapshot?") → `deleteSnapshot(id)`.
- Rename: double-click name → inline `<TextField>` → blur or Enter commits via `renameSnapshot(id, name)`.

**Deferred (YAGNI):**

- Dirty-flag indicator ("● đã chỉnh sửa"). The `loadSnapshot` confirm dialog asks unconditionally; sufficient for PR-3.1.
- Preview popover on row hover.
- Bulk delete / export.

## 10. Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| Rounding direction or margin/VAT composition diverges from legacy → user-visible price mismatches in transition period | Medium | Step 1 of the writing-plans implementation is "read `legacy.html:8550–8700` and lock the exact formula." Verified before any UI lands. |
| Performance with 14 cats × N items on every keystroke | Low (verified pattern from HotelModal) | Narrow selectors enforced in code review. LineRow takes item as prop, never subscribes to store. |
| `persist` middleware blows past localStorage quota with very large drafts | Low (one draft is <50 KB; quota ~5 MB) | `partialize` excludes `view` and `snapshots` (snapshots have separate path). |
| Template switch while a draft is dirty → user loses work | Medium | Confirm dialog before `newDraft()` if `draft.template !== null`. |
| Quote draft persists across users on a shared device | Mitigated | Per-user-scoped persist key `vte_quote_draft_${user.u}`. |
| Two browser tabs editing same draft → last-write-wins on persist | Low | Out of scope for PR-3.1. PR-3.2's cloud-history is the multi-tab sync mechanism. |
| `cloudId` shape mismatch between PR-3.1 generation and PR-3.2 consumption | Low | Algorithm lifted from `legacy.html:2677` verbatim; PR-3.2 reuses the same code path. |
| Reading large `vte_q` map on every snapshot mutation | Negligible | Map size bounded at users × 50 entries × ~5 KB each ≈ low hundreds of KB total. |
