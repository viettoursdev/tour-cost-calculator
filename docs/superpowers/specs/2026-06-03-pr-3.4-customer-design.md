## 1. Goal

Port the **Khách Hàng (Customer) tab** from the legacy monolith into the Vite app. After this PR, users can manage the customer list (create/edit/delete/search) directly from the Vite app. The legacy fallback placeholder for this tab is removed.

**Secondary benefit:** `customerStore` is now available to wire the customer-link picker into `SaveCloudQuoteModal` in a follow-up PR (PR-3.4b or later).

**Drivers:**

- The tab currently shows a "Open in legacy" placeholder — a real regression for Operations and Sales.
- Customer list is referenced from contracts (PR-3.6) and quotes (save modal). Having it ported unlocks those downstream connections.
- Self-contained: only Firestore path `viettours/customer_list`, no dependency on contracts or quotes stores.

**Non-goals:**

- Customer link in SaveCloudQuoteModal (separate follow-up).
- Customer import/export.
- Duplicate-name detection.
- Tests (refactor convention).

---

## 2. Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Firestore pattern | **Single-doc full overwrite** — `viettours/customer_list.customers[]`, same as NCC | Matches legacy `fbOnCustomers`/`fbPushCustomers`. Consistent with existing single-doc pattern. |
| State management | **Zustand `customerStore`** with `subscribeWithSelector`, no `persist` | Firestore is source of truth; no per-user localStorage needed. Same pattern as `quoteHistoryStore`. |
| Update strategy | **Optimistic local update then cloud push** | Matches legacy `handleSave` at `legacy.html:9881-9891`. UX: list updates immediately, then syncs. |
| UI layout | **Card grid** (`repeat(auto-fill, minmax(300px, 1fr))`) matching legacy | Sales already trained on this layout; `CustomerModal` as MUI `<Dialog>`. |
| Delete confirmation | **Inline MUI `<Dialog>`** (small confirm dialog) | Avoids `window.confirm` per project convention established in PR-3.2. |
| Permission guard | **`hasPerm(currentUser, 'manageCustomers')`** for create/edit/delete | Matches legacy `canEdit` prop pattern (`legacy.html:9843`). |
| Contact search | **Client-side, same filter as legacy** | Search `name`, `note`, all `contacts[*].{name,phone,email,position}`. |
| CustomerModal scope | **Create / edit / view** via `canEdit` prop | View mode (read-only) when user lacks `manageCustomers`. Same as legacy. |

## 4. Firestore API (`src/lib/firebase.ts` additions)

Document path: `viettours/customer_list` — single Firestore doc with a `customers: Customer[]` array field.

```ts
const CUSTOMER_DOC = doc(db, 'viettours', 'customer_list');

export function fbSubscribeCustomers(cb: (list: Customer[]) => void): Unsubscribe {
  return onSnapshot(CUSTOMER_DOC, (snap) => {
    cb(snap.exists() ? ((snap.data().customers as Customer[]) ?? []) : []);
  });
}

export async function fbPushCustomers(
  list: Customer[],
  pushedBy: { name: string; role: string },
): Promise<void> {
  await setDoc(CUSTOMER_DOC, {
    customers: list,
    updatedAt: new Date().toISOString(),
    updatedBy: `${pushedBy.name} (${pushedBy.role})`,
  });
}
```

## 5. `customerStore` (`src/stores/customerStore.ts`)

```ts
type CustomerState = {
  customers: Customer[];
  loading: boolean;
  syncing: boolean;

  init: () => Unsubscribe;
  save: (form: Customer) => Promise<void>;   // create or update; reads currentUser from authStore
  delete: (id: string) => Promise<void>;
};
```

**`init()`**: calls `fbSubscribeCustomers` → sets `customers` + clears `loading`. Returns `Unsubscribe`.

**`save(form)`**:
1. Derive whether create or update by `customers.find(c => c.id === form.id)`.
2. Build `next`:
   - Create: `[...customers, {...form, id: newId(), createdAt: ISO, createdBy: user.name}]`
   - Update: `customers.map(c => c.id === form.id ? {...form, updatedAt: ISO, updatedBy: user.name} : c)`
3. `set({customers: next, syncing: true})` — optimistic.
4. `await fbPushCustomers(next, {name: user.name, role: user.role})`.
5. `set({syncing: false})`.

**`delete(id)`**:
1. `const next = customers.filter(c => c.id !== id)`.
2. `set({customers: next, syncing: true})` — optimistic.
3. `await fbPushCustomers(next, ...)`.
4. `set({syncing: false})`.

Cross-store: `useAuthStore.getState().currentUser` at action time.

## 6. Component structure

```
src/components/customer/
├── CustomerView.tsx      # tab root: header + search/filter + card grid
└── CustomerModal.tsx     # MUI Dialog: create / edit / view
```

`CustomerModal` is imported and rendered inline in `CustomerView` (same pattern as `SaveCloudQuoteModal` in `QuoteView`).

### `CustomerView`

State:
- `search: string`
- `filterType: '' | 'company' | 'individual'`
- `modal: { customer: Customer | null } | null` — null = closed, `{customer: null}` = create, `{customer: c}` = edit/view
- `deleteTarget: Customer | null` — for the confirm dialog

Selectors:
- `useCustomerStore(s => s.customers)`
- `useCustomerStore(s => s.loading)`
- `useCustomerStore(s => s.syncing)`
- `useCustomerStore(s => s.save)` / `delete`
- `useAuthStore(s => s.currentUser)` → `hasPerm(currentUser, 'manageCustomers')` for `canEdit`

```
<Box p={2}>
  <Stack header>
    "👥 Danh sách Khách hàng" + count + syncing chip
    [➕ Thêm khách hàng] (if canEdit)
  </Stack>

  <Stack direction="row" search + filter Select + clear btn>

  {loading → <LinearProgress/>}
  {empty → empty state with icon}
  {data → card grid}

  {modal && <CustomerModal .../>}
  {deleteTarget && <ConfirmDeleteDialog .../>}
</Box>
```

**Customer card** (inline JSX, not a separate file):
- `<Paper variant="outlined" sx={{ cursor: 'pointer' }}>` with hover shadow.
- Type icon + name.
- Type badge chip (blue=company, green=individual).
- First 2 contacts: name · position, phone link, email link.
- "+N contact khác" if >2.
- Note preview (italic, truncated to 1 line with `overflow: hidden, textOverflow: ellipsis`).
- Edit/delete `<IconButton>` row (if canEdit), propagation-stopped.

### `CustomerModal`

Props: `{ customer: Customer | null, canEdit: boolean, onSave: (form: Customer) => void, onClose: () => void }`

Local state: `form: Customer` (initialized from `customer` prop or empty defaults).

```
<Dialog open onClose maxWidth="sm" fullWidth>
  <DialogTitle>
    {customer ? (canEdit ? '✏️ Sửa' : '👀 Xem') : '➕ Thêm'} khách hàng
  </DialogTitle>
  <DialogContent>
    <Stack spacing={2}>
      <ToggleButtonGroup exclusive value={form.type} onChange → setF('type', v) disabled={!canEdit}>
        <ToggleButton value="company">🏢 Công ty</ToggleButton>
        <ToggleButton value="individual">👤 Cá nhân</ToggleButton>
      </ToggleButtonGroup>

      <TextField label={`Tên ${form.type === 'company' ? 'công ty' : 'cá nhân'} *`}
        value={form.name} onChange → setF('name', v)
        required disabled={!canEdit} error={!form.name.trim()} />

      <ContactsSection form contacts canEdit setContact addContact delContact />

      <TextField label="Ghi chú" multiline rows={3}
        value={form.note} onChange disabled={!canEdit} />
    </Stack>
  </DialogContent>
  <DialogActions>
    <Button onClick={onClose}>Huỷ</Button>
    {canEdit && (
      <Button variant="contained" disabled={!form.name.trim()} onClick={handleSave}>
        💾 Lưu khách hàng
      </Button>
    )}
  </DialogActions>
</Dialog>
```

**ContactsSection** (local helper, not a separate file): renders the `form.contacts` array — each contact in a `<Paper>` with a 2×2 `<Grid2>` (name/position/phone/email TextFields) + delete IconButton. "➕ Thêm contact" at bottom (if canEdit).

## 8. Risks

| Risk | Mitigation |
|---|---|
| Firestore write collision (two users saving simultaneously) | Full-overwrite last-write-wins. Same risk as legacy. Not changed. |
| Empty contacts array after deleting all contacts | `save()` validates `form.name.trim()` only (matching legacy). An empty contacts array is valid — some customers are just a name + note. |
| `customers` doc grows large (hundreds of entries, many contacts) | Single doc limit ~1MB. At ~500B per customer this is ~2000 entries before concern. Monitored but not a problem for current usage. |
| `syncing: true` stuck on push failure | `finally { set({ syncing: false }) }` in both `save` and `delete`. Error surface via `window.alert` for now (consistent with QuoteToolbar's JSON import); can upgrade to Snackbar later. |
