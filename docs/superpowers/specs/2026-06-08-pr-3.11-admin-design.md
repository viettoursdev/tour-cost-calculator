# PR-3.11 Admin (UserManagement + RateCardSync) — Design

_Date: 2026-06-08_
_Phase: 3 (final tab-by-tab port) — Admin modals_
_Builds on: PR-3.8 InvoiceModal (`c80bbca`)_
_Status: Approved design, ready for implementation plan_

---

## 1. Goal

Port the final two legacy admin modals:

1. **`UserManagementModal`** (`public/legacy.html:5118–5300+`) — CEO-only. CRUD users, role + color pickers, permission-matrix display, "Reset mặc định". Push changes to Firestore via existing `authStore.saveUsers`.
2. **`RateCardSyncModal`** (`public/legacy.html:5301–5650+`) — 3-tab modal: Cloud sync status + push/pull, JSON file Export, JSON file Import with preview + confirm.

Both modals are opened from a new settings menu in the `AppShell` AppBar.

**Out of scope:**
- Tests — refactor convention.
- Replacing plaintext passwords (out of scope for the whole refactor, spec §8).

---

## 2. Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Modal triggers | New "⚙️ Settings" `IconButton` + `Menu` in `AppShell` AppBar | Centralizes both modal entries; minimal AppBar real estate. |
| User-management permission | `hasPerm(currentUser, 'manageUsers')` → CEO only | Matches legacy line 5170-5172. |
| Rate-card sync permission | Modal opens for all logged-in users; push button gated by `hasPerm(currentUser, 'syncRateCard')` | Matches legacy 5515-5522. Read-only users can still pull/export. |
| `ROLES` runtime list | New `src/auth/ROLES.ts` exporting `ROLES: readonly Role[]` + `USER_COLORS: readonly string[]` | Type union `Role` doesn't survive at runtime; the modal needs both. |
| Self-delete guard | Disable delete button when `user.u === currentUser.u` | Mirror legacy 5247-5251. |
| Last-CEO-delete guard | On delete click, reject if `users.filter(u => u.role === 'CEO').length <= 1 && user.role === 'CEO'` | Mirror legacy 5156. |
| Password storage | Plaintext (current convention) | Per refactor spec §8. |
| Username immutability after creation | Disable username input when editing | Mirror legacy 5191. |
| Push to cloud (rate card) | Bypass the 2-second `pushDebounced` and call `fbPushMasterRC` directly | The auto-sync already covers normal edits; the modal's "Đẩy lên Cloud" button is the explicit-intent path that should not be coalesced with other changes. |
| Pull from cloud (rate card) | Call `fbPullMasterRC()` → `useRateCardStore.setState({ rates })` then alert "Đã đồng bộ. Đang tải lại..." + `setTimeout(reload, 1200)` | Reload ensures all components re-render with fresh rate data. Matches legacy behavior. |
| Subscribe to cloud meta | Modal subscribes via `fbSubscribeMasterRC` for the duration it's open | Provides live "updatedBy / updatedAt" without polling. |
| Export file format | `{ _meta:{version,type,exportedAt,exportedBy,app}, hotels, visaRates, otherRates }` | Byte-equivalent to legacy 5388-5396. |
| Import validation | Require `_meta.type === 'viettours_ratecard_master'` | Match legacy 5405. |
| Import confirm | Show preview card with hotel count, hotel cities, visa countries, otherRates count → confirm dialog → apply + reload | Match legacy 5414-5421. |
| Auto-sync vs manual push interaction | Manual push winrace: writes the doc, then the existing `fbSubscribeMasterRC` echoes it back into the store (which is a no-op since state already matches). | Idempotent; no special handling needed. |

## 3. Architecture & file plan

```
src/
├── auth/
│   └── ROLES.ts                                NEW  — ROLES + USER_COLORS arrays + DEFAULT_USERS reference
├── components/
│   ├── admin/                                  NEW DIR
│   │   ├── UserManagementModal.tsx             NEW  — CRUD users
│   │   └── RateCardSyncModal.tsx               NEW  — 3-tab cloud / export / import
│   └── shell/
│       └── AppShell.tsx                        EDIT — settings IconButton + Menu + modal mounts
```

`DEFAULT_USERS` is currently defined inside `src/stores/authStore.ts` as a module-local constant. The "Reset mặc định" button needs it — exporting it would couple the modal to the store's internals. Instead, **duplicate the seed list into `src/auth/ROLES.ts`** as `DEFAULT_USERS` and import it in both places. Six rows is not a maintenance burden.

```ts
type Props = {
  open: boolean;

  currentUser: User;        // CEO — already gated by caller

```

### Local state

```ts
type FormState = {
  u: string; p: string; name: string; role: Role; color: string;
```

Source of truth for the user list is `useAuthStore((s) => s.users)`. Mutations call `useAuthStore.getState().saveUsers(next)` which persists locally and pushes to Firestore.

- Header (red gradient) — "👤 Quản lý tài khoản · Chỉ CEO mới có quyền · {n} tài khoản"
- "➕ Thêm tài khoản mới" + "↺ Reset mặc định" toolbar buttons

- User list — avatars (`name[0]` on color circle), name, role chip, "Đang đăng nhập" chip on self, Edit + Delete buttons
- Permission matrix card — small table summarizing the 7 main permissions per role

### Validation

- Username empty → `alert('Vui lòng nhập Username')`
- Username collision when adding → `alert('Username này đã tồn tại')`
- Password empty → `alert(...)`
- Display name empty → `alert(...)`
- Self-delete → button disabled
- Last-CEO-delete → `alert('Phải còn ít nhất 1 tài khoản CEO')`

## 5. RateCardSyncModal

### Props

```ts
type Props = {
  open: boolean;
  onClose: () => void;
  currentUser: User;
};
```

### Local state

```ts
const [tab, setTab] = useState<'cloud' | 'export' | 'import'>('cloud');
const [cloudData, setCloudData] = useState<RateCardDoc | null>(null);
const [cloudLoading, setCloudLoading] = useState(true);
const [cloudStatus, setCloudStatus] = useState<{ type: 'success' | 'error'; msg: string } | null>(null);
const [pushing, setPushing] = useState(false);
const [pulling, setPulling] = useState(false);
const [importPreview, setImportPreview] = useState<ImportPreview | null>(null);
const fileRef = useRef<HTMLInputElement | null>(null);
```

`useEffect` on mount: `fbPullMasterRC()` for initial load + `fbSubscribeMasterRC` for live updates while modal open. Both keyed on `open === true`.

### Tabs

#### Cloud (`tab === 'cloud'`)

- "☁️ Dữ liệu trên Cloud" card — 3-stat grid (🏨 hotel count + cities, 🛂 visa countries, 📋 otherRates count). Meta row: "Cập nhật bởi {pushedBy} · {pushedAt}".
- "💻 Dữ liệu trên máy này" card — same 3 stats from `useRateCardStore.getState().rates`.
- Status alert (`cloudStatus`).
- Action row: "📤 Đẩy lên Cloud" (gated by `syncRateCard`) + "📥 Kéo từ Cloud" (disabled while loading or when no cloud data).
- Push handler: `confirm('Đẩy rate card hiện tại lên cloud?...')` → `fbPushMasterRC(rates, pushedBy)` directly (bypass debouncer).
- Pull handler: `confirm('Kéo rate card từ cloud về máy này?...')` → `useRateCardStore.setState({ rates: cloudData })` → `setTimeout(() => window.location.reload(), 1200)`.

#### Export (`tab === 'export'`)

- 3-stat grid showing local data + "Xuất bởi {user.name} ({user.role})".
- Single button "📤 Xuất file Master Rate Card" → builds payload `{_meta, hotels, visaRates, otherRates}` → Blob → `<a download>`.

#### Import (`tab === 'import'`)

- Warning alert.
- Dashed-border file picker button → hidden `<input type="file" accept=".json">`.
- After file selected, `setImportPreview({ data, hotelCities, hotelCount, visaCountries })`.
- Preview card with stats + "Áp dụng" / "Huỷ" buttons.
- Apply handler: `confirm(...)` → `useRateCardStore.setState({ rates: data })` → `setTimeout(reload, 500)`.

## 7. Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| Concurrent edits to the user list from two CEO clients overwrite each other | Low | `fbPushUsers` is full-overwrite; matches legacy behavior. Last-write-wins is acceptable for user records. |
| `Reset mặc định` clears non-default users | Certain (intended) | Match legacy behavior; confirm prompt warns. |
| Manual push races against auto-sync debounce | Low | Manual push uses `fbPushMasterRC` directly; auto-sync subscription delivers the same payload back as a no-op. |
| Pull-from-cloud loses in-flight unsaved edits | Medium | Confirm dialog warns "rate card hiện tại sẽ bị ghi đè". Page reload completes the resync. |
| Import of malformed JSON breaks the modal | Low | Try/catch in `FileReader.onload`; alert on parse failure. |
| Permission matrix grows out of date with `PermissionKey` | Low | Source the matrix from the `PERMISSIONS` object so it stays consistent — render only the 7 primary keys (manageUsers, editRateCard, syncRateCard, exportQuote, importQuote, viewHistory, manageContracts). |
| Cloud meta type drift (legacy stored both `pushedAt`/`pushedBy` and `exportedAt`/`exportedBy`) | Medium | Display whichever exists: `cloudMeta.pushedBy ?? cloudMeta.exportedBy ?? '—'`. |
