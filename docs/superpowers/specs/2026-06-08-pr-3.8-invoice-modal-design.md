# PR-3.8 InvoiceModal — Design

_Date: 2026-06-08_
_Phase: 3 (tab-by-tab port) — Invoice export modal_
_Builds on: PR-3.7 PaymentView (`02e1cb6`)_
_Status: Approved design, ready for implementation plan_

---

## 1. Goal

Port legacy `InvoiceModal` (`public/legacy.html:4179–4307`) + `exportInvoice` PDF (`public/legacy.html:3433–3636`) into the Vite app. The Invoice export was deferred from PR-4.1 (which shipped Excel + PDF báo giá only). After this PR, the export dropdown in the Quote toolbar gets a **🧾 Invoice** entry that opens a modal collecting:

- Customer info (name *, company, phone, email)
- Language toggle (VI / EN)
- Editable payment terms (with VI/EN defaults + a "↺ Mặc định" reset)

Submit generates an A4 bilingual invoice PDF with the company header, customer + tour info, itemized table grouped by category, subtotal/margin/VAT/total rows, "Số tiền bằng chữ", payment terms, and a 2-column signature row. Last customer + last-edited terms persist to localStorage (`vte_last_customer`, `vte_payment_terms`) and pre-populate next time.

**Out of scope:**
- Invoice for DMC template — legacy hides the entry there (DMC uses its own pricing flow).
- Tests — refactor convention.

---

## 2. Decisions

| Decision | Choice | Rationale |
|---|---|---|
| PDF font | **Helvetica only** (no DejaVu, no logo image) | Matches `exportContractPDF` / `exportPaymentRequestPDF` convention. Vietnamese text written without diacritics. |
| Company constants | **Hardcoded inside `exportInvoicePDF.ts`** | One call site, never changes; pulling into a shared module is premature. |
| Modal state | **Local `useState` in modal** + localStorage persist on submit (`vte_last_customer`, `vte_payment_terms`) | No store needed — the data is per-export, not shared. |
| Customer source | **Free-text fields** (no customer-picker from `customerStore`) | Legacy is free-text. Linking to the customer store would change the data model; deferred. |
| Lang default | `'vi'` | Matches legacy `useState("vi")`. |
| Payment terms default | VI text on first open, swap to EN defaults when user clicks `↺ Mặc định` while `lang === 'en'` | Matches legacy lines 4185-4186, 4256. |
| Trigger location | **`QuoteToolbar.tsx` export menu**, inserted between `📄 PDF báo giá` and the JSON `Divider` | Legacy exposes it in the same dropdown (legacy line 8740). |
| DMC visibility | **Hide the Invoice menu item when `template === 'dmc'`** | Legacy hides it too — DMC has separate pricing UX. |
| Permission gate | `hasPerm(user, 'exportQuote')` reuses existing permission | Matches PR-4.1 pattern. |
| Numbers | All amounts in **VND only**; English version says "VND" rather than the local symbol | Matches legacy `Amount in words: ${grandTotal.toLocaleString("en-US")} VND.` |
| Amount in words | `numberToVietWords` from `src/lib/exports/vietWords.ts` for VI · plain English number for EN | Same helper PR-3.6 / PR-3.7 use. |
| File naming | `Invoice_{invNo}_{customerSlug}.pdf` where `invNo = VTE-YYYYMMDD-HHmm`, `customerSlug = customer.name.replace(/[^a-zA-Z0-9_]/g,'_')` | Exact legacy match (line 3635). |

## 4. PDF signature

```ts
export interface InvoiceCustomer {
  name: string;
  company: string;
  phone: string;
  email: string;
}

export function exportInvoicePDF(args: {
  draft: QuoteDraft;
  totals: Totals;                       // from computeTotals(draft)
  customer: InvoiceCustomer;
  lang: 'vi' | 'en';
  paymentTerms: string;                 // newline-separated; empty = use defaults
  savedBy: { name: string };            // currentUser for sign-off row
}): void;
```

Note: takes `totals` rather than recomputing — they're already on screen, the caller already has them. Avoids duplicating `computeTotals` plumbing.

## 5. Modal props

```ts
type Props = {
  open: boolean;
  onClose: () => void;
  draft: QuoteDraft;
  totals: Totals;
  user: User;
};
```

Local state:
- `customer: InvoiceCustomer` — hydrated from `localStorage.vte_last_customer` on first open; reset to defaults when localStorage absent.
- `lang: 'vi' | 'en'`
- `paymentTerms: string` — hydrated from `localStorage.vte_payment_terms` on first open; otherwise VI defaults.
- `editTerms: boolean` — toggles the terms panel between read-only preview and `<TextField multiline>`.

Submit handler:
1. Persist `customer` → `vte_last_customer`.
2. Persist `paymentTerms` → `vte_payment_terms`.
3. Call `exportInvoicePDF({ draft, totals, customer, lang, paymentTerms, savedBy: { name: user.name } })`.
4. `onClose()`.

Submit button disabled until `customer.name.trim()`.

## 6. Trigger wiring (QuoteToolbar)

After the `📄 PDF báo giá` menu item (`QuoteToolbar.tsx` lines 195-203 of the current file) and **before** the `<Divider />`, insert:

```tsx
{draft.template && draft.template !== 'dmc' && currentUser && (
  <MenuItem onClick={() => { setInvoiceOpen(true); setExportAnchor(null); }}>
    <ListItemIcon><ReceiptIcon fontSize="small" /></ListItemIcon>
    <ListItemText>🧾 Invoice</ListItemText>
  </MenuItem>
)}
```

QuoteToolbar holds the modal open state via a new `const [invoiceOpen, setInvoiceOpen] = useState(false);` and renders `<InvoiceModal open={invoiceOpen} ... />` at the end of the component (sibling to existing modals).

## 7. Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| Helvetica can't render Vietnamese diacritics | Certain | Strip diacritics in the PDF text (matches `exportContractPDF` convention). Modal UI keeps full Vietnamese for screen display. |
| Long item names overflow the description column | Medium | Use `pdf.splitTextToSize` per legacy line 3531; only render line 0 (matches legacy clipping). |
| Multi-page invoices break the footer | Low | Footer loop iterates `pdf.internal.getNumberOfPages()` per legacy 3624-3633. |
| Customer name with slashes / unicode escapes file save | Low | Sanitize with `replace(/[^a-zA-Z0-9_]/g, '_')` per legacy. |
| `paymentTerms` containing literal `\n` (legacy bug at line 3592 splits on `"\\n"`) | Low | Fix in port — split on `'\n'` not the escaped `"\\n"`. This matches what users actually type into the textarea. |

## 9. Manual verification

- **V1:** From a non-DMC draft, open the export menu → `🧾 Invoice` is visible; from DMC it is hidden.
- **V2:** Click it → modal opens with VI defaults, customer fields pre-filled from localStorage if previously saved.
- **V3:** Toggle EN; click `↺ Mặc định` → terms swap to EN defaults.
- **V4:** Type a custom term; submit → `Invoice_VTE-*.pdf` downloads; localStorage shows `vte_payment_terms` updated.
- **V5:** Open the modal again → custom term persists; customer fields persist.
- **V6:** Submit with empty `customer.name` → submit button disabled.
- **V7:** PDF: company header, BILL TO + TOUR INFO blocks, itemized table grouped by category with alt-row shading, subtotal + margin + VAT + TOTAL, "Số tiền bằng chữ" (VI) or "Amount in words" (EN), payment terms, signature row, footer with page numbers.
- **V8:** Items with `foc: true` render as `FOC` in green in the Amount column.
