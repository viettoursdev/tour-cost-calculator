# PR-3.3 Dashboard — Design

_Date: 2026-06-03_
_Phase: 3 (tab-by-tab port)_
_Builds on: PR-3.1 + PR-3.2 (Quote core + cloud history, merged at `61ab751`)_
_Status: Approved design, ready for implementation plan_

---

## 1. Goal

Port the legacy `DashboardView` — a margin / profit analysis view of the **currently active quote draft** — into the Vite app as the 4th view-toggle inside the Quote tab. Remove the broken top-level "Dashboard" placeholder tab from `AppShell`.

**Drivers:**

- Closes a real (small) regression vs legacy: Sales loses the per-category margin breakdown.
- Cleans up an artifact of the original tab list (the top-level `'dashboard'` placeholder links nowhere in legacy and confuses users who click it).
- Smallest PR in Phase 3 — pure read-only computation, reuses everything.

**Non-goals:**

- Cross-quote / multi-draft aggregations (recent quotes, monthly totals, etc.). Defer until contracts + customers ported.
- Chart libraries (no recharts / chart.js).
- A standalone "company overview" top-level surface.

---

## 2. Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Dashboard scope | **Per-quote margin breakdown only** (matches legacy `DashboardView`) | Avoids inventing product surfaces. Legacy users get strict parity. |
| Surface | **4th toggle in the Quote toolbar** (`Chi phí · Tổng kết · Dashboard · Lịch sử`) | Mirrors legacy where Dashboard is a Quote sub-view, not a top-level tab. |
| Top-level tab cleanup | **Remove `'dashboard'` from `AppShell` `TABS`** | The placeholder linked to `/legacy.html#tab=dashboard` which doesn't exist in legacy either. |
| Layout fidelity | **Mirror legacy aesthetic** — gradient KPI cards, hero banner, category breakdown grid, full-vs-per-pax summary | Sales already trained on this layout; MUI `<Paper>` + `sx` gradients reproduce cleanly. |
| Data source | **Local draft only** (via `quoteStore`) | No Firestore subscription. No new store. |
| Computation | **Reuse `computeTotals()` from PR-3.1's `calc.ts`** | Same totals math as the cost-totals sidebar in `CostView` and the summary in `SummaryView`. One source of truth. |
| Per-category margin allocation | `marginAlloc = (catTotal / totalCost) × totalProfit` | Lifted verbatim from `legacy.html:4027`. |
| Per-category profit % | `marginAlloc / (catTotal + marginAlloc) × 100` | Lifted verbatim from `legacy.html:4028`. |
| Tests | Out of scope | Matches refactor convention. |

## 4. Component shape

### `DashboardView.tsx`

Subscribes to the same narrow draft slices as `CostView` (template / items / catEnabled / pax / rates / margin / vat / svcBasis / rounding). Computes:

1. `totals = computeTotals({...draft})` from `calc.ts` — gives totalCost / totalProfit / totalVAT / sellingPPax / roundedPPax / grandTotal.
2. `catTotals = getCATS(template).map(c => ({...c, total: catEnabled[c.id] ? catTotal(items[c.id] ?? [], rates, pax) : 0, enabled: catEnabled[c.id]}))` — per-category VND totals.
3. Per category: `costPct = total / totalCost × 100`, `marginAlloc = (total / totalCost) × totalProfit`, `profitPct = marginAlloc / (total + marginAlloc) × 100`.

### Render sections

**A. Hero banner** (legacy lines 3994-3997):
- `<Paper>` with `sx={{background: 'linear-gradient(135deg, #0d7a6a, #14a08c)', color: '#fff', p: 3, borderRadius: 2}}`.
- Title: "📊 Phân tích biên lợi nhuận".
- Subtitle: `Chi phí {pax} khách → Giá bán {fmtVND(roundedPPax)}/khách`.

**B. 4 KPI cards** (legacy lines 3999-4020):

```
┌─────────────────────────────────────────────────────────────────┐
│ Tổng chi phí gốc │ Service Charge {margin}% │ VAT {vat}% │ Giá bán cả đoàn │
│ {totalCost}      │ {totalProfit}            │ {totalVAT} │ {grandTotal}    │
│ {x}/khách        │ {x}/khách                │ {x}/khách  │ {roundedPPax}/khách │
└─────────────────────────────────────────────────────────────────┘
```

Each card: `<Paper>` with gradient bg, `sx={{p: 2, color: '#fff', borderRadius: 2}}`. Container: `<Box display="grid" gridTemplateColumns="repeat(auto-fit, minmax(220px, 1fr))" gap={2}>`.

Gradient palette (lifted from legacy):
- Cost: `#3498db → #2980b9` (blue)
- Service Charge: `#9b59b6 → #8e44ad` (purple)
- VAT: `#f5a623 → #e67e22` (orange)
- Selling: `#dc3250 → #c0392b` (red)

**C. Per-category breakdown** (legacy lines 4022-4058):
- Outer: `<Paper variant="outlined" sx={{p: 2.5, borderRadius: 2}}>`.
- Title: "💰 Phân tích chi phí từng hạng mục".
- Grid of cat cards (one per enabled cat with `total > 0`): `<Box display="grid" gridTemplateColumns="repeat(auto-fit, minmax(260px, 1fr))" gap={1.5}>`.
- Each cat card: subtle tinted bg derived from `cat.color`, shows cat icon + label + 4 stat rows (cost / % of total / margin alloc / margin %).

**D. Full vs per-pax summary** (legacy lines 4060-4094):
- Outer: `<Paper variant="outlined" sx={{p: 2.5, borderRadius: 2}}>`.
- Title: "📈 Tóm tắt lợi nhuận".
- 2-column grid: "Cả đoàn ({pax} khách)" | "/ Khách". Each column has 4 rows: cost / service charge / VAT / selling, with the last row styled bigger / heavier weight.

### Selector hygiene

Same projection pattern as `CostView`:

```tsx
const template = useQuoteStore(s => s.draft.template);
const items = useQuoteStore(s => s.draft.items);
const catEnabled = useQuoteStore(s => s.draft.catEnabled);
const pax = useQuoteStore(s => s.draft.pax);
const rates = useQuoteStore(s => s.draft.rates);
const margin = useQuoteStore(s => s.draft.margin);
const vat = useQuoteStore(s => s.draft.vat);
const svcBasis = useQuoteStore(s => s.draft.svcBasis);

const draftSnapshot = { template, info: {...}, pax, rates, margin, vat, svcBasis, rounding, items, catEnabled, currentQuoteId: null };

```

## 6. Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| User had clicked into the top-level Dashboard tab and now finds it gone | Low — placeholder was never functional | Mention in PR description; no migration needed since the tab linked nowhere useful. |
| Margin-allocation formula drift vs legacy | Negligible | Formula lifted verbatim from `legacy.html:4027-4028`. Same `computeTotals()` source as CostView/SummaryView. |
| Gradient KPI cards look out-of-place vs MUI flat theme | Low | Constrained to 4 cards in this view only; legacy aesthetic. |
| Adding a 4th toggle crowds the toolbar on narrow viewports | Low | `ToggleButtonGroup` collapses gracefully; `useFlexGap` already in place. |
