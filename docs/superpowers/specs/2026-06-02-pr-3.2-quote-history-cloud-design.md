# PR-3.2 Quote History Cloud — Design

_Date: 2026-06-02_
_Phase: 3 (tab-by-tab port)_
_Builds on: PR-3.1 Quote core (merged at `9d7157b`)_
_Status: Approved design, ready for implementation plan_

---

## 1. Goal

Add **Firestore-synced cross-device quote history** to the Vite app. After this PR, users can save the active draft to a shared cloud library, share quotes with collaborators (filtered by visibility), and load any visible cloud quote back into the editor — eliminating the only meaningful regression the Vite app still has versus the legacy app's Quote tab.

**Drivers:**

- PR-3.1 shipped per-user, per-device local snapshots. Users lose cross-device save/load that legacy provided.
- Sales depends on collaborator sharing (Trưởng Phòng adds collaborators on quotes for Operations review). Currently still forced to legacy for any cloud workflow.
- Cuts another meaningful chunk of legacy traffic now that the Quote editor is fully ported.

**Non-goals:**

- **DMC view + DMC cloud paths** (PR-3.9). PR-3.2 ships regular quotes only (`viettours/quote_history` + `quote_projects/{id}`). The Vite app's template selector still greys out DMC, so users can't *create* DMC quotes from Vite anyway. DMC quotes saved from legacy remain editable in legacy until PR-3.9.
- **Customer link in SaveCloudQuoteModal** — preserved on legacy-originated entries, but no picker in the Vite Save modal until PR-3.4 (Customer tab) ships.
- **Version restore UI.** Versions are *saved* (matching legacy `fbSaveQuoteState` semantics, 20 per quote FIFO) so legacy users can restore via legacy; the Vite app only loads `currentState`. Add restore UI in a follow-up if users ask.
- **Permission-gated "view all quotes"** override for CEO/Trưởng Phòng. Legacy doesn't do this; Vite matches.
- **Optimistic locking / conflict resolution.** Last-write-wins per project convention; matches legacy.
- **Real-time collaborator-edit indicators** ("Linh is editing this quote").
- **Push notifications on collaborator-add** (PR-3.10).

## 2. Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Scope: DMC | **Regular only.** No `dmc_quote_history` / `dmc_quote_projects` paths. | Symmetric with PR-3.1's scope. PR-3.9 adds DMC. |
| UI surface | **New `'history'` value in the Quote toolbar's view switcher** (`Chi phí ↔ Tổng kết ↔ Lịch sử`). Full-page `QuoteHistoryView` rendered in the main area; right-side `<HistPanel/>` drawer for local snapshots stays. | Familiar to legacy users. Plenty of room for metadata columns. Clear mental separation: drawer = personal quick saves, full view = shared cloud library. |
| Save modal | **Separate `SaveCloudQuoteModal`** from PR-3.1's local-snapshot Save dialog. Fields: `name`, `collaborators` (multi-select Autocomplete over `authStore.users`), optional `versionNote`. | Local saves are quick/private; cloud saves are deliberate/shared. Different fields, different semantics. Matches legacy's two-modal split. |
| Customer link | **Deferred.** Existing cloud entries' `customerId/customerName` fields are preserved when re-saving from Vite; legacy users can still set them. PR-3.4 adds the picker. | Avoids coupling PR-3.2 to PR-3.4. Customer-less Vite saves leave the field empty — legacy renders that fine. |
| Version history | **Save versions, no restore UI.** `fbSaveQuoteState` appends each save as a new version (max 20). Vite reads `currentState` only. | Restoring an old version is rare; preserving versions keeps cross-app compat without spending UI surface. |
| Table component | **`@mui/x-data-grid`** for `QuoteHistoryView`. | Fixed columns, no in-cell editing, free sort/filter/keyboard nav. Bundle cost already paid (mui chunk). Better fit here than for HotelModal's compound cells (PR-2.1). |
| Visibility filter | **Client-side: `createdByUsername === user.u OR collaborators.some(c => c.u === user.u)`.** Computed on read. | Matches legacy. Firestore rules stay open per project convention (no `request.auth` since custom auth). Visibility is presentation, not security. |
| List cap | **500 entries** (FIFO eviction on save). | Matches legacy `fbSaveQuote` `.slice(0, 500)`. |
| Version cap | **20 per quote** (FIFO eviction). | Matches legacy `fbSaveQuoteState` `.slice(-20)`. |
| Quote code | **Lifted verbatim** from legacy `generateQuoteCode` (`legacy.html:235-248`). Format `NĐ.01.31.05.26` / `NN.01.31.05.26`. | Same code on either app's save. |
| Collaborator UX | **Picker in SaveCloudQuoteModal** for set-on-save. **Popover triggered from row 👥 button** for edit-in-place. Both use the same `Autocomplete` over `authStore.users`. | Save modal handles initial set; popover handles "add Hùng now that I've onboarded him." |
| Conflict on simultaneous edits | **Last-write-wins, no warning.** | Matches legacy. Acceptable for a small Sales team where coordination is human. |
| Tests | **Out of scope.** | Matches whole-refactor convention. |

**Boundary discipline:**

- `quoteStore.saveCloud` writes both `quote_history` AND `quote_projects/{cloudId}` — atomic at the function boundary, not at the Firestore boundary (best-effort).
- Components subscribe via narrow selectors. `LineRow` / `CatBlock` / `CostView` etc. from PR-3.1 are unchanged.

## 4. Data layer

### 4.1 Types added to `src/types/quote.ts`

```ts
export type Collaborator = {
  u: string;          // username
  name: string;       // display name at the time of add
};

export type QuoteVersion = {
  versionNo: number;
  savedAt: string;    // ISO
  savedBy: string;    // "Tony (CEO)"
  note: string;
  state: QuoteDraft;
};

export type CloudQuoteEntry = {
  id: number;                  // Date.now() at first save
  cloudId: string;             // Date.now().toString(36) + random(3) — Firestore doc path
  quoteCode: string;           // "NĐ.01.31.05.26"
  name: string;
  template: Template;
  pax: number;
  totalCost: number;
  customerId?: string;         // preserved on Vite saves; populated by legacy or PR-3.4+
  customerName?: string;
  createdByUsername: string;
  createdByName: string;
  collaborators: Collaborator[];
  createdAt: string;
  updatedAt: string;
  updatedBy: string;
};

export type CloudQuoteProject = {
  versions: QuoteVersion[];    // max 20
  currentState: QuoteDraft;
  collaborators: Collaborator[];
  updatedAt: string;
  updatedBy: string;
};
```

**Compat note:** `CloudQuoteEntry.id` is the same shape as `Snapshot.id` from PR-3.1 (numeric `Date.now()`). `cloudId` is a separate string ID used as the Firestore `quote_projects/{cloudId}` document path. PR-3.1's `Snapshot.cloudId` already generates this format; PR-3.2 simply reuses it for the first cloud save.

### 4.2 Firestore lib (`src/lib/firebase.ts` additions)

Document paths:

- `viettours/quote_history` — single doc with `quotes: CloudQuoteEntry[]` (cap 500, FIFO).
- `quote_projects/{cloudId}` — one doc per quote: `{versions[], currentState, collaborators[], updatedAt, updatedBy}`.

Function surface (signatures shown; bodies lift verbatim from `legacy.html:160-341`):

```ts
export function fbSubscribeQuoteHistory(cb: (quotes: CloudQuoteEntry[]) => void): Unsubscribe;

export async function fbSaveQuote(
  entry: { id: number; cloudId: string; quoteCode?: string; name: string; template: Template; pax: number; totalCost: number; customerId?: string; customerName?: string; collaborators?: Collaborator[] },
  savedBy: { u: string; name: string; role: string },
): Promise<CloudQuoteEntry>;

export async function fbSaveQuoteState(
  cloudId: string,
  state: QuoteDraft,
  note: string | undefined,
  savedBy: { name: string; role: string },
): Promise<void>;

export async function fbDeleteQuote(id: number, cloudId: string): Promise<void>;

export async function fbUpdateCollaborators(
  id: number,
  cloudId: string,
  collaborators: Collaborator[],
): Promise<void>;

export async function fbGetQuoteProject(cloudId: string): Promise<CloudQuoteProject | null>;

export function generateQuoteCode(template: Template, existing: CloudQuoteEntry[]): string;
```

**`fbSaveQuote` semantics** (lifted from `legacy.html:307-335`):

- If `quotes.findIndex(q => q.id === entry.id) >= 0`: merge into existing entry, preserve `createdByUsername`/`createdByName`/`createdAt`, update `updatedAt`/`updatedBy`, keep existing `collaborators` unless `entry.collaborators` is explicitly provided.
- Else: create new entry. Set creator info from `savedBy`, generate `quoteCode` if missing, initialize `createdAt`/`updatedAt`/`updatedBy`, `collaborators ?? []`.
- Always `.slice(0, 500)` and write back.

**`fbSaveQuoteState` semantics** (lifted from `legacy.html:164-176`):

- Read existing project doc (or `{versions: [], collaborators: []}` if missing).
- `versionNo = (existing.versions?.length ?? 0) + 1`.
- New version `{versionNo, savedAt: ISO, savedBy: "Tony (CEO)", note: note ?? "Phiên bản {N}", state}`.
- `versions = [...existing.versions, newVersion].slice(-20)`.
- Write `{...existing, versions, currentState: state, updatedAt: ISO, updatedBy: savedBy.name}`.

**`fbUpdateCollaborators` semantics** (lifted from `legacy.html:181-193`):

- Write `collaborators` into `quote_projects/{cloudId}` (merged with existing).
- Also write into `viettours/quote_history.quotes[idx].collaborators` so visibility filter picks it up immediately.

**`fbDeleteQuote` semantics:**

- Filter the entry out of `viettours/quote_history.quotes[]`.
- Best-effort `deleteDoc(quote_projects/{cloudId})` — log a warning on failure, do not throw (legacy doesn't delete project docs either, but Vite cleans them up to avoid orphans).

**`generateQuoteCode`:** lifted verbatim from `legacy.html:235-248`. Format `NĐ.01.31.05.26` (domestic), `NN.01.31.05.26` (intl), `DMC.01.31.05.26` (dmc). Seq is per-day-per-prefix count from `existing`.

### 4.3 `quoteHistoryStore` (`src/stores/quoteHistoryStore.ts`)

```ts
type QuoteHistoryState = {
  quotes: CloudQuoteEntry[];         // raw list from Firestore (unfiltered)
  loading: boolean;
  error: string | null;
  init: (user: User) => Unsubscribe;
  visibleQuotes: () => CloudQuoteEntry[];
};
```

- `init(user)` subscribes via `fbSubscribeQuoteHistory((quotes) => set({quotes, loading: false}))`. Returns the unsubscribe.
- `visibleQuotes()` filters by `createdByUsername === currentUser.u OR collaborators.some(c => c.u === currentUser.u)`. Computed on call.
- `error` set if `fbSubscribeQuoteHistory`'s observer fires `next(error)`. Components show inline.

Not wrapped in `persist` — Firestore is the source of truth.

### 4.4 `quoteStore` additions

```ts
saveCloud: (name: string, collaborators: Collaborator[], note?: string) => Promise<CloudQuoteEntry>;
deleteCloud: (id: number, cloudId: string) => Promise<void>;
updateCloudCollaborators: (id: number, cloudId: string, collabs: Collaborator[]) => Promise<void>;
loadCloud: (cloudId: string) => Promise<{ ok: true } | { ok: false; error: string }>;
```

`saveCloud` flow:

1. `totalCost = computeTotals(draft).totalCost`
2. `cloudId = draft.currentQuoteId ?? Date.now().toString(36) + random3()` — reuses if already linked to a cloud quote.
3. `isNew = !draft.currentQuoteId`. If new, `quoteCode = generateQuoteCode(draft.template!, quoteHistoryStore.getState().quotes)`. Otherwise leave undefined — `fbSaveQuote` preserves the existing code.
4. `await fbSaveQuote({id, cloudId, quoteCode, name, template: draft.template!, pax: draft.pax, totalCost, collaborators}, currentUser)`.
5. `await fbSaveQuoteState(cloudId, draft, note, currentUser)`.
6. `set({draft: {...draft, currentQuoteId: cloudId}})`.
7. Return the entry.

`loadCloud`: `fbGetQuoteProject(cloudId)`; if null, return `{ok: false, error: 'Báo giá không tồn tại trong cloud'}`. Else `set({draft: {...project.currentState, currentQuoteId: cloudId}, view: 'cost'})`.

`deleteCloud`: `fbDeleteQuote(id, cloudId)`. If the deleted quote is `draft.currentQuoteId`, also `set({draft: {...draft, currentQuoteId: null}})`.

`updateCloudCollaborators`: `fbUpdateCollaborators(id, cloudId, collabs)`. No local state mutation needed — Firestore round-trip updates `quoteHistoryStore`.

`view` union becomes `'cost' | 'summary' | 'history'`.

## 6. Save flow + visibility filter (detailed)

### Save sequence

```
User clicks [Lưu cloud ☁️] in QuoteToolbar
  └─ SaveCloudQuoteModal opens
     │  name = draft.info.name (editable)
     │  collaborators = [] (Vite app does not auto-include current user — Sales adds explicitly)
     │  note = ""
     │  existingQuoteCode = quotes.find(q => q.cloudId === draft.currentQuoteId)?.quoteCode
     └─ User fills, clicks [Lưu mới] or [Cập nhật]
        └─ quoteStore.saveCloud(name, collaborators, note)
           ├─ totalCost = computeTotals(draft).totalCost
           ├─ cloudId = draft.currentQuoteId ?? newId()
           ├─ if !draft.currentQuoteId: quoteCode = generateQuoteCode(draft.template!, history.quotes)
           ├─ await fbSaveQuote({...}, currentUser)
           │     → Firestore write → onSnapshot fires → history.quotes updates
           ├─ await fbSaveQuoteState(cloudId, draft, note, currentUser)
           │     → new version appended (cap 20)
           ├─ set({draft: {...draft, currentQuoteId: cloudId}})
           └─ return entry
        └─ Modal closes; Snackbar "✅ Đã lưu lên cloud"
```

**Error handling:** if either `fb*` call rejects, the catch in `SaveCloudQuoteModal.confirmSave` surfaces an inline `<Alert severity="error">` with the error message, leaves the modal open, does NOT close, does NOT update `currentQuoteId`. User can retry. Partial state (entry exists but no project doc) self-heals on retry.

### Visibility

```ts
visibleQuotes: () => {
  const u = useAuthStore.getState().currentUser;
  if (!u) return [];
  return get().quotes.filter((q) =>
    q.createdByUsername === u.u || q.collaborators.some((c) => c.u === u.u),
  );
}
```

- Client-side. Firestore rules stay open per project convention.
- No role override (CEO sees same as everyone). Document; revisit if requested.
- `QuoteHistoryView` wraps in `useMemo([quotes, currentUser?.u])`.

### Quote code generation

Lifted from `legacy.html:235-248`. `NĐ.01.31.05.26` format. Seq is `pad(existing.filter(prefix.startsWith && date.endsWith).length + 1, 2)`.

## 7. Edge cases

| Case | Behavior |
|---|---|
| User saves, closes tab, reopens → draft restored from `vte_quote_draft_${user.u}` | `currentQuoteId` is in the persisted draft. Re-saves update the existing entry. |
| User loads cloud quote A, edits, then loads cloud quote B without saving | Confirm dialog "Báo giá hiện tại có thay đổi chưa lưu. Tiếp tục?" Same pattern as PR-3.1 HistPanel. |
| Two users edit the same cloud quote simultaneously | Last-write-wins. Both writes succeed; `onSnapshot` shows the other's writes after they land. Matches legacy. |
| `fbSaveQuote` succeeds, `fbSaveQuoteState` fails | Metadata entry exists, project doc missing. Retry on next save heals it. `loadCloud(cloudId)` returns `{ok: false}` until then. |
| `onSnapshot` callback fires before `currentUser` is set in auth | `visibleQuotes()` returns `[]`. UI shows empty list briefly. |
| Collaborator removed from a quote they had loaded | Their `draft.currentQuoteId` still points at it; subsequent save still works (rules open). On next QuoteHistoryView render the quote disappears from their list. Acceptable for PR-3.2. |
| User deletes a cloud quote that's `draft.currentQuoteId` | Confirm. After delete: `set({draft: {...draft, currentQuoteId: null}})` so next save creates new. Draft content stays in editor. |
| Imported JSON file has `currentQuoteId` field | Preserved on import. Re-saving updates that cloud entry; "Báo giá mới" abandons the link. |
| 500-quote cap reached | Legacy silently drops oldest. Match. |
| Save while `draft.template === null` | Cannot happen — Lưu cloud button is rendered only inside the `template !== null` branch of QuoteView. |
| User edits collaborators via popover but loses network | `fbUpdateCollaborators` rejects. Popover surfaces error; keeps the local picker open. Retry on user action. |

## 9. Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| `viettours/quote_history` doc shape drifts between legacy and Vite | Low | `fbSaveQuote` lifts the legacy semantics verbatim (merge-vs-create branching, preserve createdBy on updates, init defaults). Smoke test before push: save from Vite → reload in legacy. |
| `quote_projects/{cloudId}` `currentState` shape mismatches legacy reader | Medium | `QuoteDraft` types match what legacy `MainApp` stores. Smoke test: save Vite → open in legacy → confirm it loads cleanly. |
| Vite `currentQuoteId` (string) vs legacy assumption | Low | Same generation algorithm (`Date.now().toString(36) + random(3)`), same shape used as Firestore doc id. |
| Performance with 500 entries × Firestore round-trip on every save | Low | DataGrid handles 500 rows easily; `onSnapshot` payload <500 KB; debounce not needed. |
| User accidentally shares a quote with someone outside their team | Low — picker shows full user list | Acceptable for internal tool (8 roles total, all internal). Document. |
| Cloud-saved DMC quotes from legacy show up in Vite list | Possible (filter is template-agnostic) | **DMC entries ARE in `dmc_quote_history`, not `quote_history`** — they're in a SEPARATE Firestore doc. PR-3.2 subscribes to `quote_history` only. DMC quotes therefore won't appear. ✓ |
| Concurrent saves of two different quotes by the same user race the 500-cap | Low | `slice(0, 500)` is applied after `.unshift` — write-then-cap is server-side. Worst case: lose one of two simultaneously-pushed-out-of-frame entries. Acceptable. |
