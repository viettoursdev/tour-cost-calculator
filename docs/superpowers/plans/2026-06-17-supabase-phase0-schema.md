# Supabase Migration — Phase 0 (Schema) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the complete normalized Postgres schema (≈40 tables), RLS policies, the first-login provisioning trigger, and the Realtime publication on a local + cloud Supabase project, validated by pgTAP tests — the standalone foundation every later phase builds on.

**Architecture:** Introduce a `supabase/` directory managed by the Supabase CLI. Each table family is one timestamped migration file with a paired pgTAP test file under `supabase/tests/`. Schema is derived column-by-column from `src/types/*.ts`. RLS is parity-only: every table requires an authenticated user whose JWT `email` ends in `@viettours.com.vn`. A trigger on `auth.users` INSERT creates a default `profiles` row. No application code (`src/`) changes in this phase.

**Tech Stack:** Supabase CLI, PostgreSQL 15 (Supabase managed), pgTAP (`supabase test db`), SQL migrations. Docker required for local Supabase.

## Global Constraints

- **No `src/` changes in Phase 0.** This phase only adds `supabase/` and CI/docs. The Vite app build, ESLint, and `tsc` must remain green and untouched. Do **not** add `@supabase/supabase-js` to `package.json` yet (Phase 1).
- **Postgres target: 15** (Supabase default). Use `gen_random_uuid()` (pgcrypto/pgcrypto built-in in PG15) for UUID defaults.
- **Identity is canonical UUID.** Every "who created this" reference is `created_by uuid REFERENCES profiles(id)` **nullable**, paired with a `created_by_name text` display column. User-list references (collaborators, staff, thread members) are `uuid` FKs to `profiles(id)`.
- **RLS parity rule (verbatim):** policy predicate is `(auth.jwt() ->> 'email') ILIKE '%@viettours.com.vn'`. Applied as `USING` (read) and `WITH CHECK` (write) on every table. No role logic in SQL.
- **Value-blob rule (verbatim from spec):** sub-structures with identity worth querying → child tables; scalar config / opaque maps → JSONB columns; version snapshots → one row with a `state`/`sections`/`products` JSONB column.
- **Rate card is opaque.** `HotelEntry`/`OtherRateEntry`/`VisaRates` are `Record<string, unknown>` in `src/types/rates.ts` — store as JSONB, never column-shredded.
- **Commits:** Conventional Commits. One migration family per commit. Co-author line: `Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>`.
- **Bootstrap CEO email:** `developer@viettours.com.vn` (matches `authStore` `BOOTSTRAP_CEO_EMAIL`). Store as a Postgres setting read by the trigger, not hardcoded in the policy.

Create `supabase/migrations/0001_profiles_and_provisioning.sql`:
```sql
-- Canonical user profile, 1:1 with auth.users.
create table public.profiles (
  id         uuid primary key references auth.users(id) on delete cascade,
  username   text unique,
  email      text,
  role       text not null default 'Standard'
             check (role in ('CEO','Ban Giám Đốc','Trưởng Phòng','Sales',
                             'Operations','Marketing','Admin','Accountant','Standard')),
  name       text not null default '',
  color      text not null default '#888888',
  phone      text,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

alter table public.profiles enable row level security;

create policy profiles_company_read  on public.profiles for select using (public.is_viettours_user());
create policy profiles_company_write on public.profiles for all
  using (public.is_viettours_user()) with check (public.is_viettours_user());

-- Bootstrap CEO email (read by the trigger; not hardcoded in policies).
alter database postgres set app.bootstrap_ceo_email = 'developer@viettours.com.vn';

-- First-login provisioning: create a default profile for every new auth user.
create or replace function public.handle_new_user()
returns trigger
language plpgsql
security definer
set search_path = public
as $$
declare
  ceo_email text := current_setting('app.bootstrap_ceo_email', true);
begin
  insert into public.profiles (id, email, username, name, role)
  values (
    new.id,
    new.email,
    split_part(new.email, '@', 1),
    split_part(new.email, '@', 1),
    case when new.email = ceo_email then 'CEO' else 'Standard' end
  )
  on conflict (id) do nothing;
  return new;
end;
$$;

create trigger on_auth_user_created
  after insert on auth.users
  for each row execute function public.handle_new_user();
```
