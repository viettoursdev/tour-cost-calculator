## Task 1: Port `exportInvoicePDF`

**Files:**
- Create: `src/lib/exports/exportInvoicePDF.ts`

- [ ] **Step 1:** Define `InvoiceCustomer` type and the `exportInvoicePDF(args)` signature from spec §4.
- [ ] **Step 2:** Hardcode the company constants `BRAND = 'VIETTOURS INCENTIVES & EVENTS'`, tax code `0302650371`, address `19B Mai Thi Luu, P. Tan Dinh, TP. Ho Chi Minh`.
- [ ] **Step 3:** Port the full A4 layout from legacy 3433-3636. Helvetica only — no `addImage` for the logo. Use ASCII-stripped Vietnamese (same style as `exportContractPDF`, `exportPaymentRequestPDF`).
- [ ] **Step 4:** Reuse `numberToVietWords` from `vietWords.ts` for the Vietnamese "Bằng chữ" line; English line uses `.toLocaleString('en-US')` + " VND".
- [ ] **Step 5:** Iterate items via the active categories (use `getCATS(template)` filter passed via `draft.catEnabled`). Reuse `calcVND` from `@/components/quote/calc`. Stamp `FOC` (in green) when `item.foc === true`.
- [ ] **Step 6:** Totals rows pull from `args.totals` (avoid recomputing). Show: subtotal+svcBasis, profit, VAT, then the rounded `grandTotal` row in red bold.
- [ ] **Step 7:** Payment-terms split on `'\n'` (not `'\\n'` — fix the legacy bug). When `paymentTerms` is blank, fall back to the per-language defaults.
- [ ] **Step 8:** Multi-page footer loop iterates `pdf.internal.getNumberOfPages()`.
- [ ] **Step 9:** `pdf.save(\`Invoice_${invNo}_${customerSlug}.pdf\`)` where `invNo = VTE-YYYYMMDD-HHmm`, `customerSlug = customer.name.replace(/[^a-zA-Z0-9_]/g,'_')`.
- [ ] **Step 10:** `npm run typecheck && npm run lint`.
- [ ] **Commit:** `feat(exports): exportInvoicePDF (PR-3.8 Task 1)`
