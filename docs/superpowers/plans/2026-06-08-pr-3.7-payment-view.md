# PR-3.7 PaymentView Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` or `superpowers:executing-plans`. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Port the per-tour payment tracker (legacy `PaymentView` at `public/legacy.html:4311–4675`) into the Vite app as a sub-view inside the Quote tab, **including the 2-stage approval flow** (PaymentRequestModal, PDF export, NotificationBell stage progression).

**Architecture:** Extend `QuoteState.view` with `'payment'`. Add a new `paymentStore` keyed by `tourKey` (slug of `info.name`) with lazy Firestore subscriptions and 1s-debounced writes mirroring legacy `vte_payments_{tourKey}` / `tour_payments/{tourKey}` shapes. Build `PaymentView` + two modals (`TrackItemsModal`, `AddCustomCostModal`) and a shared `paymentUtils.ts`. Add a `Thanh toán` toggle to `QuoteToolbar` (hidden for DMC).

**Tech Stack:** TypeScript · React 18 · Vite · MUI v6 · Zustand · Firebase Firestore (named DB `viettours`).

**Spec:** `docs/superpowers/specs/2026-06-08-pr-3.7-payment-view-design.md`

**Tests:** Out of scope per spec §11. Each task verifies via `npm run typecheck` and `npm run lint`. Final task is full manual verification in the browser.

**Conventions per task:**
- Run `npm run typecheck && npm run lint` after every code change.
- Commit at the end of each task. Commit messages follow Conventional Commits (`feat(payment): …`, `refactor(firebase): …`).
- Co-author every commit: `Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>`.

Wrap each parse in try/catch — corrupt entries become empty defaults.
- [ ] **Step 3:** `ensureSubscribed(tourKey)` — if slot doesn't exist, create one with empty data. If `slot.unsub === null`, call `fbSubscribeTourPayments(tourKey, data => { ... })`. - [ ] **Step 4:** `releaseSubscription(tourKey)` — decrement; if `refCount` reaches 0 call `slot.unsub?.()` and set `slot.unsub = null`.
- [ ] **Step 5:** `setPayments(tourKey, next)` and `setCustomItems(tourKey, next)` — update slot in place, write localStorage synchronously, then schedule a debounced push: `clearTimeout(slot.pushTimer); slot.pushTimer = setTimeout(() => fbSaveTourPayments(tourKey, slot.data.payments, slot.data.customItems, savedBy).catch(()=>{}), 1000);`. Resolve `savedBy` from `useAuthStore.getState().currentUser` at flush time.
- [ ] **Step 6:** `getTour(tourKey)` — return `slots[tourKey]?.data ?? { payments: {}, customItems: [] }`.
