## 2. Module đã rà — xác nhận SẠCH (không cần sửa)

| Module | Vì sao yên tâm |
|---|---|
| **Kho (Inventory)** | Xuất FIFO (`inventory_issue` RPC): `pg_advisory_xact_lock` chống đua, kiểm đủ tồn trước khi trừ (chặn âm), `order by received_at` (FIFO), ghi đúng `unit_cost` mỗi movement; `inventory_adjust` chặn số âm |
| **Quyết toán / Tạm ứng đa ngoại tệ** | `advanceCalc.ts` dùng `rate>0` (không giấu chi phí), balance dư→hoàn/thiếu→công nợ đúng; `computeSettlement` guard chia-0 |
| **Link báo giá công khai** | Token `crypto.getRandomValues` 96-bit, đọc qua RPC token-required (chống enumerate), accept-once gated, revoke = DELETE (đọc trả rỗng); payload **ẩn giá vốn/margin** — chỉ field hướng khách |
| **Thông báo / Worker** | Dedup localStorage bị chặn kích thước; `daysUntil` đúng cho VN (UTC+7); worker **verify chữ ký JWT thật** (ES256/RS256) fail-closed; digest sáng chống chạy trùng 12h |
| **Nghỉ phép** | `leaveDayCount` đếm bao trùm (+1), guard chặn âm, date-only UTC không lệch timezone |
| **Lịch HDV (pure logic)** | `detectConflicts` bắc cầu chặng dài, loại nối-chuyến cùng booking, overlap vs buffer, vắt-giao-thừa đúng |
| **Chấm công** | Tính công thuần + persistence per-row + import cảnh báo trước khi đè + khoá + reset xác nhận khi sửa |

## 3. Checklist vận hành

### Sau mỗi lần push có `supabase/migrations/**`
- [ ] Kiểm run **`migrate.yml`**: `gh run list --workflow=migrate.yml --limit 5`.
- [ ] Nếu run ở trạng thái **`waiting`** → cổng `production-db` cần duyệt tay:
  ```bash
  RID=<run id>
  gh api repos/viettoursdev/tour-cost-calculator/actions/runs/$RID/pending_deployments   # lấy env id (production-db = 17080910499)
  gh api repos/viettoursdev/tour-cost-calculator/actions/runs/$RID/pending_deployments \
     -X POST -f state=approved -F 'environment_ids[]=17080910499' -f comment='<lý do>'
  ```
- [ ] Xác nhận log: `Applying migration <n>… Finished supabase db push.`
- [ ] Cân nhắc bật **Required reviewers** cố định cho env `production-db` (an toàn hơn tự-áp), vì hiện hành vi cổng lúc tự-áp lúc chặn.

### Trước mỗi push
- [ ] `npm run build` (typecheck + vite) xanh — pre-push hook đã chạy typecheck.
- [ ] `npm run lint` (`--max-warnings 0`) xanh.
- [ ] `npm test` (unit) xanh; integration `npm run test:integration` nếu đụng gateway/DB.
- [ ] Push bằng tài khoản `viettoursdev`:
  `git -c credential.helper='!gh auth git-credential' push origin HEAD:main`

### Nguyên tắc dữ liệu (BẮT BUỘC cho code mới)
- [ ] Bảng chia sẻ (RLS `for all`): **KHÔNG** dùng delete-diff/full-overwrite. Luôn **upsert-only + targeted delete** riêng.
- [ ] Cột **uuid FK** trỏ `customers(id)`: resolve legacy→uuid trước khi ghi **và** uuid→legacy khi đọc (hoặc dùng cột `text` lưu legacy như `tour_profiles`).
- [ ] Field mới của `QuoteDraft`: nhớ round-trip qua `quote_versions.state` (snapshot đầy đủ đã tự lo, nhưng field cần dùng ở shredded thì thêm vào `decomposeQuote`/`assembleQuote`).
- [ ] PDF khách: tránh glyph ngoài Latin (`→`, emoji) với font VN subset — dùng `safeArrow`/ASCII.
- [ ] Migration: chỉ **ADD** nullable, không destructive; backup R2 theo giờ là lưới an toàn cuối.

### Nhiều phiên/người sửa song song
- [ ] Chỉ `git add` **file của mình** (KHÔNG `git add -A`) — nhiều feature chạy chung working tree.
- [ ] Một vùng code nên do **một** phiên sở hữu tại một thời điểm để tránh trùng/xung đột.
